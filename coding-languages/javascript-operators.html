Arithmetic Operators

Arithmetic operators in JavaScript are used to perform mathematical calculations like addition, subtraction, multiplication, and more. They are the foundation for working with numbers in JavaScript.

List of Arithmetic Operators

  • + (Addition): Adds two numbers together.
  • - (Subtraction): Subtracts the second number from the first.
  • * (Multiplication): Multiplies two numbers.
  • / (Division): Divides the first number by the second.
  • % (Modulus): Returns the remainder of a division.
  • ** (Exponentiation): Raises the first number to the power of the second.
  • ++ (Increment): Increases a number by one.
  • -- (Decrement): Decreases a number by one.

Examples

let a = 10;
let b = 3;

console.log(a + b); // Output: 13 (Addition)
console.log(a - b); // Output: 7 (Subtraction)
console.log(a * b); // Output: 30 (Multiplication)
console.log(a / b); // Output: 3.333... (Division)
console.log(a % b); // Output: 1 (Modulus)
console.log(a ** b); // Output: 1000 (Exponentiation)

let x = 5;
x++; // Increment
console.log(x); // Output: 6
x--; // Decrement
console.log(x); // Output: 5

Special Notes

  • Division: Dividing by zero in JavaScript results in Infinity.
  • Modulus: Works with both integers and floating-point numbers.
  • Increment and Decrement: Can be used as pre-increment (++x) or post-increment (x++), which affects when the value changes in an expression.

Best Practices

  • Use parentheses to clarify the order of operations, especially in complex expressions.
  • Be cautious with division and modulus when working with floating-point numbers, as results may not be exact.
  • Always initialize variables before using increment or decrement operators.

Comparison Operators

Comparison operators in JavaScript are used to compare two values. These comparisons always return a Boolean result: true or false. They are essential for making decisions in your code using conditionals, such as if statements.

List of Comparison Operators

  • == (Equal To): Compares two values for equality, but does not check the data type.
  • === (Strict Equal To): Compares two values for equality and also checks the data type.
  • != (Not Equal To): Compares two values for inequality, ignoring data type.
  • !== (Strict Not Equal To): Compares two values for inequality and checks the data type.
  • > (Greater Than): Checks if the left value is greater than the right value.
  • < (Less Than): Checks if the left value is less than the right value.
  • >= (Greater Than or Equal To): Checks if the left value is greater than or equal to the right value.
  • <= (Less Than or Equal To): Checks if the left value is less than or equal to the right value.

Examples

let a = 5;
let b = "5";
let c = 10;

console.log(a == b); // Output: true (Values are equal, but types are ignored)
console.log(a === b); // Output: false (Values are equal, but types are different)
console.log(a != c); // Output: true (5 is not equal to 10)
console.log(a !== b); // Output: true (Values are equal, but types are different)

console.log(c > a); // Output: true (10 is greater than 5)
console.log(c < a); // Output: false (10 is not less than 5)
console.log(c >= a); // Output: true (10 is greater than or equal to 5)
console.log(a <= b); // Output: true (5 is less than or equal to "5")

Special Notes

  • == vs ===: Always prefer === (strict equality) to avoid unexpected results due to type coercion.
  • != vs !==: Use !== (strict inequality) for the same reason as above.
  • Comparison with strings: JavaScript compares strings lexicographically (based on Unicode values).

Best Practices

  • Always use strict equality (===) and strict inequality (!==) unless type coercion is intentional.
  • Use parentheses in complex conditions to clarify precedence.

Logical Operators

Logical operators in JavaScript are used to combine multiple conditions or to invert the logic of a condition. These operators are often used in conditional statements like if, while, and for loops.

List of Logical Operators

  • && (Logical AND): Returns true if both conditions are true.
  • || (Logical OR): Returns true if at least one of the conditions is true.
  • ! (Logical NOT): Inverts the truth value of the condition.

Examples

let a = true;
let b = false;

// Logical AND
console.log(a && b); // Output: false (both conditions must be true)

// Logical OR
console.log(a || b); // Output: true (only one condition needs to be true)

// Logical NOT
console.log(!a); // Output: false (inverts true to false)
console.log(!b); // Output: true (inverts false to true)

Combining Logical Operators

Logical operators can be combined to form complex conditions. For example:

let age = 25;
let hasPermission = true;

// Check if the age is within a range and permission is granted
if (age >= 18 && age <= 30 && hasPermission) {
  console.log("Access granted.");
} else {
  console.log("Access denied.");
}

Truthiness and Falsiness

JavaScript treats certain values as "truthy" or "falsy" in logical expressions:

  • Falsy values: false, 0, "" (empty string), null, undefined, and NaN.
  • Truthy values: Everything else, including {} (empty object) and [] (empty array).
console.log(Boolean(0)); // Output: false
console.log(Boolean("hello")); // Output: true
console.log(Boolean([])); // Output: true

Special Notes

  • Short-Circuit Evaluation: Logical operators stop evaluating as soon as the result is determined. For example:
    console.log(false && someFunction()); // Output: false (does not call someFunction())
    console.log(true || someFunction()); // Output: true (does not call someFunction())
  • Use parentheses: To clarify precedence when combining multiple logical operators.

Assignment Operators

Assignment operators in JavaScript are used to assign values to variables. They can also perform an operation (like addition or multiplication) and assign the result to the variable in one step.

List of Assignment Operators

  • = (Assignment): Assigns a value to a variable.
  • += (Addition Assignment): Adds a value to the current value of the variable and assigns the result.
  • -= (Subtraction Assignment): Subtracts a value from the current value of the variable and assigns the result.
  • *= (Multiplication Assignment): Multiplies the current value of the variable by a value and assigns the result.
  • /= (Division Assignment): Divides the current value of the variable by a value and assigns the result.
  • %= (Modulus Assignment): Takes the remainder when the current value of the variable is divided by a value and assigns the result.
  • **= (Exponentiation Assignment): Raises the current value of the variable to the power of a value and assigns the result.

Examples

let x = 10;

// Basic Assignment
x = 5;
console.log(x); // Output: 5

// Addition Assignment
x += 3;
console.log(x); // Output: 8

// Subtraction Assignment
x -= 2;
console.log(x); // Output: 6

// Multiplication Assignment
x *= 4;
console.log(x); // Output: 24

// Division Assignment
x /= 3;
console.log(x); // Output: 8

// Modulus Assignment
x %= 5;
console.log(x); // Output: 3

// Exponentiation Assignment
x **= 2;
console.log(x); // Output: 9

Combining Assignment Operators

Assignment operators allow you to write shorter, more concise code. For example:

let total = 0;
total += 10; // Adds 10
total -= 3; // Subtracts 3
console.log(total); // Output: 7

Special Notes

  • Assignment operators are right-associative, meaning the expression is evaluated from right to left. For example:
    let a = b = 10;
    console.log(a); // Output: 10
    console.log(b); // Output: 10
  • Always initialize variables before using assignment operators to avoid errors.

Bitwise Operators

Bitwise operators in JavaScript operate directly on the binary representation of numbers. They are primarily used in low-level programming tasks, such as working with flags, masks, or performance-critical code.

List of Bitwise Operators

  • & (AND): Sets each bit to 1 if both corresponding bits are 1.
  • | (OR): Sets each bit to 1 if at least one of the corresponding bits is 1.
  • ^ (XOR): Sets each bit to 1 if the corresponding bits are different.
  • ~ (NOT): Inverts all the bits (0 becomes 1, and 1 becomes 0).
  • << (Left Shift): Shifts bits to the left, filling with 0s, and effectively multiplying the number by 2 for each shift.
  • >> (Right Shift): Shifts bits to the right, preserving the sign bit (for signed numbers).
  • >>> (Zero-Fill Right Shift): Shifts bits to the right, filling with 0s regardless of the sign.

Examples

let a = 5; // Binary: 0101
let b = 3; // Binary: 0011

console.log(a & b); // Output: 1 (Binary: 0001)
console.log(a | b); // Output: 7 (Binary: 0111)
console.log(a ^ b); // Output: 6 (Binary: 0110)
console.log(~a); // Output: -6 (Inverts bits: 1010 becomes -6 in 2's complement)

console.log(a << 1); // Output: 10 (Binary: 1010, 5 shifted left by 1)
console.log(a >> 1); // Output: 2 (Binary: 0010, 5 shifted right by 1)
console.log(a >>> 1); // Output: 2 (Binary: 0010, same as above for unsigned numbers)

Bitwise vs Logical Operators

While bitwise operators work on individual bits of numbers, logical operators work with entire Boolean expressions. For example, & is bitwise AND, whereas && is logical AND.

Applications of Bitwise Operators

  • Setting or clearing specific bits in a number (using masks).
  • Efficiently checking whether a number is even or odd (using & 1).
  • Optimizing certain mathematical or logical computations.

Special Notes

  • Bitwise operators treat numbers as 32-bit signed integers internally.
  • Be cautious with ~, as it inverts all bits, including the sign bit, potentially leading to negative results.

Ternary Operators

The ternary operator is a concise way to write conditional statements in JavaScript. It evaluates a condition and returns one of two values based on whether the condition is true or false.

Syntax

condition ? valueIfTrue : valueIfFalse;

The condition is evaluated first. If it is true, the expression after the ? is returned. If it is false, the expression after the : is returned.

Examples

let age = 18;
let canVote = (age >= 18) ? "Yes" : "No";
console.log(canVote); // Output: "Yes"

let number = 5;
let isEven = (number % 2 === 0) ? "Even" : "Odd";
console.log(isEven); // Output: "Odd"

Chaining Ternary Operators

Ternary operators can be chained for more complex conditions, but be cautious as it can affect readability:

let score = 85;
let grade = (score >= 90) ? "A" : (score >= 80) ? "B" : "C";
console.log(grade); // Output: "B"

Best Practices

  • Use the ternary operator for simple conditions to keep your code concise.
  • Avoid chaining multiple ternary operators if it reduces code readability.
  • For complex conditions, consider using an if-else statement instead.

Common Use Cases

  • Setting default values based on a condition.
  • Rendering different outputs in UI frameworks like React.
  • Quick decision-making in one-liners.

What is the difference between == and === in JavaScript?

The == operator compares two values for equality but ignores their data types (type coercion occurs). The === operator compares both the value and the data type (strict equality).

console.log(5 == "5"); // Output: true (type coercion: "5" is converted to a number)
console.log(5 === "5"); // Output: false (value is the same, but types are different)

Write a code snippet to demonstrate the use of != and !==. What is the difference between them?

The != operator checks if two values are not equal, ignoring their data types. The !== operator checks if two values are not equal and also considers their data types.

console.log(5 != "5"); // Output: false (type coercion: "5" is treated as a number)
console.log(5 !== "5"); // Output: true (value is the same, but types are different)

Explain what happens when you compare two strings in JavaScript. How are they evaluated?

Strings in JavaScript are compared lexicographically (alphabetically), based on their Unicode values. For example:

Write a chained ternary operator to assign a grade (A, B, or C) based on the following conditions:

  • A: Score is 90 or above
  • B: Score is between 80 and 89
  • C: Score is below 80
