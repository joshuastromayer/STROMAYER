If, Else, and Else If

Conditional statements allow your code to make decisions and execute different blocks of code based on specific conditions. The if, else, and else if statements are fundamental tools for control flow in JavaScript.

If Statement

The if statement executes a block of code only if a specified condition is true.

let age = 18;
if (age >= 18) {
  console.log("You are eligible to vote.");
}

If-Else Statement

The else statement specifies a block of code to execute if the condition in the if statement is false.

let age = 16;
if (age >= 18) {
  console.log("You are eligible to vote.");
} else {
  console.log("You are not eligible to vote.");
}

If-Else If Statement

The else if statement allows you to test multiple conditions sequentially. Once a condition evaluates to true, the corresponding block of code is executed, and the rest are skipped.

let score = 85;
if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 80) {
  console.log("Grade: B");
} else {
  console.log("Grade: C");
}

Comparison Operators in Conditions

Conditions often use comparison operators. Here are some examples:

  • == and ===: Checks equality.
  • > and <: Checks greater than or less than.
  • >= and <=: Checks greater than or equal to, less than or equal to.
let num = 10;
if (num % 2 === 0) {
  console.log("The number is even.");
} else {
  console.log("The number is odd.");
}

Best Practices

  • Always use === for strict equality checks to avoid unexpected type coercion.
  • Use else only when necessary to handle default cases.
  • Keep conditions simple and easy to understand.

Switch Statements

Switch statements in JavaScript are used for decision-making and provide an alternative to multiple if/else if statements when dealing with a single variable or expression. They allow you to test a variable against multiple possible values and execute specific code based on the match.

Syntax

switch (expression) {
  case value1:
    // Code to execute if expression === value1
    break;
  case value2:
    // Code to execute if expression === value2
    break;
  default:
    // Code to execute if no case matches
}

The break statement prevents the code from "falling through" to the next case. The default block is optional and executes if no matching case is found.

Example 1: Basic Usage

Here's an example where we use a switch statement to determine the day of the week:

const day = 3;
let dayName;

switch (day) {
  case 1:
    dayName = "Monday";
    break;
  case 2:
    dayName = "Tuesday";
    break;
  case 3:
    dayName = "Wednesday";
    break;
  case 4:
    dayName = "Thursday";
    break;
  case 5:
    dayName = "Friday";
    break;
  default:
    dayName = "Weekend";
}

console.log(dayName); // Output: Wednesday

Example 2: Falling Through

In JavaScript, if you omit the break statement, the code will "fall through" to the next case:

const color = "yellow";

switch (color) {
  case "red":
    console.log("Red");
  case "yellow":
    console.log("Yellow");
  case "green":
    console.log("Green");
    break;
  default:
    console.log("Unknown color");
}

// Output:
// Yellow
// Green

Notice how Yellow and Green are both logged because there is no break after the case "yellow".

Best Practices

  • Always use break unless you intentionally want the code to fall through to the next case.
  • Use the default block to handle unexpected values.
  • Consider using if/else if statements for complex conditions or ranges.

When to Use a Switch Statement

A switch statement is best used when:

  • You are testing a single variable or expression against multiple possible values.
  • The conditions are straightforward and non-overlapping.
  • You want to improve readability compared to multiple if/else if statements.

Ternary Operator

The ternary operator is a concise way to write conditional statements in JavaScript. It's often used as a shorthand for simple if...else statements. The ternary operator uses the following syntax:

condition ? expressionIfTrue : expressionIfFalse;

Here's how it works:

  • If the condition evaluates to true, the expressionIfTrue is executed.
  • If the condition evaluates to false, the expressionIfFalse is executed.

Example: Checking Eligibility

Suppose you want to determine if a person is old enough to vote:

const age = 18;
const canVote = (age >= 18) ? "Yes, eligible to vote" : "No, not eligible to vote";
console.log(canVote); // Outputs: "Yes, eligible to vote"

Chaining Ternary Operators

You can chain ternary operators for multiple conditions, but it can reduce code readability if overused. Here's an example:

const score = 85;
const grade = (score >= 90) ? "A" : (score >= 80) ? "B" : (score >= 70) ? "C" : "F";
console.log(grade); // Outputs: "B"

Note: If the logic gets too complex, it's better to use if...else for clarity.

Use Cases

The ternary operator is great for situations where you need to assign values conditionally. Some common use cases include:

  • Setting default values:
    • const name = userInput ? userInput : "Guest";
    console.log(name); // Outputs the userInput value or "Guest" if userInput is falsy.
  • Inline conditional rendering in frameworks like React.

Advantages

  • Concise and readable for simple conditions.
  • Can be used inline, making it useful for quick assignments.

Disadvantages

  • Can become difficult to read if overused or chained extensively.
  • Not suitable for complex conditions involving multiple statements.

Nested Conditionals

Sometimes in programming, we need to evaluate multiple conditions that are dependent on each other. Nested conditionals are a way to use one conditional statement inside another to create more complex decision-making structures. While this can be powerful, it should be used with caution to avoid overly complex and hard-to-read code.

What Are Nested Conditionals?

Nested conditionals occur when an if, else if, or else block contains another if statement. This allows for more granular checks within a condition.

Basic Example

const age = 18;
const hasLicense = true;

if (age >= 18) {
  console.log("You are old enough to drive.");
  if (hasLicense) {
    console.log("You are allowed to drive!");
  } else {
    console.log("You need a license to drive.");
  }
} else {
  console.log("You are too young to drive.");
}

In this example, the program first checks if the person is old enough to drive. If so, it then checks if they have a license.

Practical Use Case

Imagine an online store where you want to apply a discount based on the customer's membership status and their total purchase amount:

const isMember = true;
const totalAmount = 150;

if (isMember) {
  if (totalAmount > 100) {
    console.log("You get a 20% discount!");
  } else {
    console.log("You get a 10% discount!");
  }
} else {
  if (totalAmount > 100) {
    console.log("You get a 5% discount!");
  } else {
    console.log("No discount available.");
  }
}

In this example, nested conditionals allow for specific actions based on membership and purchase amount.

Tips for Using Nested Conditionals

  • Keep them readable: Avoid excessive nesting, as it can make the code hard to follow.
  • Use functions: When nested conditionals become too complex, consider refactoring parts of the code into functions.
  • Comment your code: Leave comments to explain the logic behind the nested structure.

Common Mistakes

  • Not using else blocks: Ensure that all potential conditions are handled to avoid unexpected behavior.
  • Overusing nested conditionals: Too much nesting can lead to code that's hard to debug and maintain.

Short-Circuiting

Short-circuiting is a concept in JavaScript where logical operators like && (AND) and || (OR) stop evaluating as soon as the result of the expression is determined. This behavior can be leveraged to write more efficient and concise code.

Logical Operators and Short-Circuiting

  • && (AND): Evaluates to true only if all conditions are true. If the first condition is false, it stops and returns false.
  • || (OR): Evaluates to true if any condition is true. If the first condition is true, it stops and returns true.

Examples

Using &&

const user = { name: "Alice", isLoggedIn: true };
user.isLoggedIn && console.log("Welcome, " + user.name);

// Output: "Welcome, Alice"
// If isLoggedIn is false, the second condition is not executed.

Using ||

const userName = "";
const displayName = userName || "Guest";
console.log(displayName);

// Output: "Guest"
// If userName is empty, the default "Guest" is returned.

Common Use Cases

  • Default Values: Use || to assign default values when variables are falsy (e.g., undefined, null, false, "", 0).
  • Conditional Execution: Use && to execute code only when a condition is true.

Important Notes

  • The || operator stops at the first truthy value, while the && operator stops at the first falsy value.
  • Be cautious when using short-circuiting with non-boolean values, as the results might be unexpected in certain cases.

Example: Combining Short-Circuiting

const isMember = true;
const discount = isMember && "20%";
console.log(discount); // Output: "20%"

const userInput = "";
const output = userInput || "Default value";
console.log(output); // Output: "Default value"

Short-Circuiting: Exercises

1. What is short-circuiting in JavaScript, and which logical operators exhibit this behavior?

Short-circuiting in JavaScript is the behavior where logical operators stop evaluating as soon as the result of the expression is determined. The operators that exhibit this behavior are && (AND) and || (OR).

2. Explain how the && operator short-circuits with an example.

The && operator short-circuits when the first condition is false, as the entire expression cannot be true. For example:

const isAdult = false;
isAdult && console.log("You are eligible to vote.");

// Output: Nothing is logged, as the first condition is false.

3. Explain how the || operator short-circuits with an example.

The || operator short-circuits when the first condition is true, as the entire expression is already true. For example:

const userName = "Alice";
const displayName = userName || "Guest";
console.log(displayName); // Output: "Alice"

4. What will be the output of the following code?

const user = isLoggedIn && "Welcome, User";
console.log(user);

Output:

false, because the first condition isLoggedIn is false, and the second part of the expression is not evaluated.

const userInput = "";
