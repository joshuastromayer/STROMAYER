JS Comments

Comments in JavaScript are lines of text that the JavaScript engine ignores during execution. They are useful for explaining code, making notes, and temporarily disabling parts of a script.

Why Use Comments?

Comments are essential for writing readable and maintainable code. They help:

  • Explain complex logic to yourself and others.
  • Make code easier to understand and debug.
  • Temporarily disable specific lines of code for testing.

1. Single-Line Comments

Single-line comments start with // and extend to the end of the line.

<script>
// This is a single-line comment
console.log("Hello, World!"); // This comment is ignored
</script>

2. Multi-Line Comments

Multi-line comments start with /* and end with */. They span multiple lines.

<script>
/*
This is a multi-line comment.
It can span multiple lines.
*/
console.log("Multi-line comments are useful!");
</script>

3. Using Comments for Debugging

You can use comments to temporarily disable parts of your code for debugging.

<script>
let message = "Hello, World!";
// console.log(message); // This line is now ignored
</script>

Best Practices for Writing Comments

  • Use comments to explain why something is done, not just what it does.
  • Avoid excessive comments on simple code (e.g., // Add 2 + 2 is unnecessary).
  • Use proper formatting and spacing to make comments clear and readable.
